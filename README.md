# portal-web-hub

## 项目简介

本仓库用于归档和发布多个独立 HTML 页面。这些页面以静态资源的形式存储，可通过 GitHub Pages 等托管服务进行访问。仓库不针对任何特定域名或网站，仅作为页面集合的存储与分发点。

## 目录说明

```
portal-web-hub/
├── pages/            # 存放各 HTML 页面文件及其附属资源
├── assets/           # 公共资源文件（样式、脚本、图片等）
├── index.html        # 入口页面（可选）
└── README.md         # 本文件
```

- `pages/`：每个子目录或文件对应一个独立页面，命名清晰以避免冲突。
- `assets/`：存放多个页面共用的样式表、JavaScript 文件或图片，便于统一管理。
- `index.html`（可选）：若启用 Pages 服务，可作为默认首页。

## 页面归档说明

归档的 HTML 页面均为独立构建，不依赖外部数据库或后端服务。每个页面通常包含完整的 HTML、CSS 和 JavaScript，可直接在浏览器中打开运行。部分页面可能引用 `assets/` 目录下的公共资源。

### 添加新页面

1. 将页面文件放入 `pages/` 目录下。
2. 如有需要，将公共资源放入 `assets/` 目录并在页面中正确引用路径。
3. 提交 Pull Request 或直接推送至主分支。

### 页面命名规范

- 使用小写英文字母、数字和连字符（`-`）。
- 避免使用特殊字符和空格。
- 命名应反映页面内容或其功能。

## 维护说明

- 本仓库由贡献者共同维护，欢迎提交改进或修复。
- 提交前请确保页面在主流浏览器中正常显示。
- 不建议在页面中包含外部不可控资源（如第三方 CDN），以免因资源失效导致页面异常。
- 仓库不存储敏感信息或用户数据，所有页面内容应保持公开、合规。
